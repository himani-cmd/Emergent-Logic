# Social Creative Queue

Prepared: 2026-06-29  
Owner: Social Creative Agent  
Scope: LinkedIn profile, LinkedIn company page, Instagram  

## Brand Guardrails

- Use navy, white, muted blue, soft gray, and clean spacing.
- Use the EL circle mark as the main brand mark.
- Avoid loud gradients, neon colors, generic SaaS shapes, emoji-heavy captions, and hype.
- Keep the language practical, operator-led, and specific.
- Reuse the same idea across channels only when the format changes meaningfully.

## Recommended Template Types

- Lead Leak Card: simple headline, small CRM handoff diagram, EL circle mark.
- CRM Cleanup Checklist: before/after table, duplicate/contact/owner/source fields.
- Workflow Map Portrait: vertical lead journey from form to CRM to follow-up.
- Case Study Proof Card: method-based proof, not client-result claims.
- Operator Note Card: calm quote-style insight from Himani.
- Mini Audit Carousel: 5 to 7 slides with one practical check per slide.

## 4. HubSpot Lifecycle Stages

Channel: LinkedIn profile  
Posted by: Himani profile  
Recommended template type: Operator Note Card  
Graphic concept: Clean lifecycle ladder with Lead, MQL, SQL, Opportunity, Customer. Add a small warning label: "Only useful if the rules are clear."  
CTA: Ask whether their HubSpot lifecycle stages are clear.

Post copy:

HubSpot lifecycle stages are useful only when everyone knows what they mean.

If "Lead" means one thing to marketing and another thing to sales, reports become hard to trust.

If MQL is updated manually, it gets missed.

If SQL does not mean a real sales conversation happened, pipeline becomes inflated.

The fix is usually not complicated.

Define the stages.
Define the trigger for each stage.
Clean old records.
Automate only the parts that are safe to automate.

HubSpot is much easier to use when the basic definitions are not fuzzy.

## 6. AI Automation Guardrails

Channel: Instagram  
Posted by: Company  
Recommended template type: Mini Audit Carousel  
Graphic concept: 5-slide carousel with a navy cover: "AI automation should remove a bottleneck." Slides compare good use cases vs weak use cases.  
CTA: Save this checklist before building an automation.

Post copy:

AI automation should not be added because it sounds modern.

It should remove a real bottleneck.

Good use cases:

- summarize incoming inquiries
- draft follow-up notes
- tag requests by type
- create CRM tasks
- detect missing information
- route leads to the right owner

Weak use cases:

- automating unclear processes
- sending generic replies
- hiding messy CRM data
- replacing basic follow-up discipline

Start with the workflow.

Then decide where AI actually helps.
